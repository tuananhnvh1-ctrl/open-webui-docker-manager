# Open WebUI Docker Manager

A beginner-friendly desktop manager for deploying and managing Open WebUI with Docker.

This app provides a CustomTkinter GUI around common Docker workflows: deploy, redeploy, start, stop, restart, remove, inspect logs, and open Open WebUI in the browser.

## Features

- One-click deployment for Open WebUI Docker images
- Standard CPU, NVIDIA CUDA, bundled Ollama, and custom presets
- Docker Run and Docker Compose command preview
- Persistent Docker volume support
- Optional `OLLAMA_BASE_URL` configuration
- Optional `host.docker.internal:host-gateway` mapping
- Health/readiness checks through Docker status and `/health`
- Live log following without freezing the UI
- Windows launcher with automatic virtual environment setup

## Requirements

- Python 3.10 or newer
- Docker Desktop on Windows/macOS, or Docker Engine on Linux
- Internet access for the first package/image download
- Optional: NVIDIA GPU + NVIDIA Container Toolkit for CUDA mode

## Quick Start on Windows

1. Install Docker Desktop and start it.
2. Install Python 3.10+ and make sure `python` is available in Command Prompt.
3. Download or clone this repository.
4. Double-click `run.bat`.
5. Select a preset and click **Deploy / Redeploy**.
6. Open `http://localhost:3000` after the app reports that WebUI is ready.

The launcher creates `.venv`, installs dependencies, writes `launcher.log`, and runs `main.py`.

## Manual Start

```bash
python -m venv .venv
.venv\Scripts\python.exe -m pip install --upgrade pip
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe main.py
```

On Linux/macOS:

```bash
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip
.venv/bin/python -m pip install -r requirements.txt
.venv/bin/python main.py
```

## Presets

### Standard CPU

Uses:

```text
ghcr.io/open-webui/open-webui:main
```

This is the recommended default when Ollama runs on the host machine.

### NVIDIA GPU

Uses:

```text
ghcr.io/open-webui/open-webui:cuda
```

Only enable this if your system has an NVIDIA GPU and the NVIDIA Container Toolkit is installed.

### Bundled Ollama CPU

Uses:

```text
ghcr.io/open-webui/open-webui:ollama
```

This runs Open WebUI and Ollama in the same container and persists both Open WebUI data and Ollama model data in Docker volumes.

## Configuration

The app stores GUI settings in:

```text
~/.open-webui-manager/config.json
```

Docker Compose files generated by the app are written to:

```text
~/.open-webui-manager/
```

## Notes for GitHub Publishing

Recommended repository name:

```text
open-webui-docker-manager
```

Use only Latin letters, numbers, hyphens, or underscores in the project path. This reduces encoding issues in Windows batch files and Python virtual environments.

Before publishing:

1. Replace the copyright holder in `LICENSE`.
2. Add screenshots to the README if you want.
3. Test `run.bat` on a clean Windows machine.
4. Confirm Docker Desktop is running before demoing the app.

## Troubleshooting

### Docker unavailable

Start Docker Desktop, then click **Check Docker** in the app.

### Permission denied on Linux

Add your user to the Docker group, then log out and back in:

```bash
sudo usermod -aG docker $USER
```

### Open WebUI is not ready immediately

The first start can take 25-60 seconds or longer because the container may run migrations and initialize the database. Use **Check WebUI Connection** or the **Logs** tab.

### Port already in use

Change the host port from `3000` to another free port, such as `3001`.
